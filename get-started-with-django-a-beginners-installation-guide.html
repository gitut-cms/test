Get Started with Django: A Beginner's Installation Guide

Published on April 06, 2025 | Category: jjj

So, you've heard about Django, the high-level Python web framework that encourages rapid development and clean, pragmatic design. Built by experienced developers, it takes care of much of the hassle of web development, so you can focus on writing your application without needing to reinvent the wheel.

Ready to dive in and start building? The first step is getting Django installed on your system. Don't worry, it's straightforward! This guide will walk you through the process step-by-step.

Before We Begin: Prerequisites

Django is a Python framework, so naturally, you'll need Python installed.

  1. Python: Django requires Python. Check if you have it installed by opening your terminal or command prompt and typing:

    python --version
# or on some systems
python3 --version

    If you see a version number (Django currently supports Python 3.8+ - check the official Django docs for the specific versions supported by the latest Django release), you're good to go. If not, head over to python.org and download the latest stable version for your operating system.

  2. pip: This is Python's package installer. It usually comes bundled with Python versions 3.4 and later. You can check if you have it (and its version) with:

    pip --version
# or
pip3 --version

    If you need to install or upgrade pip, follow the instructions on the official pip installation guide. It's generally a good idea to have the latest version:

    pip install --upgrade pip

Step 1: Create and Activate a Virtual Environment (Highly Recommended!)

Before installing Django, it's best practice to create a dedicated virtual environment for your project. Why?

  • Isolation: It keeps the dependencies required by different projects separate. Installing Django globally can lead to version conflicts if another project needs a different version.
  • Cleanliness: Your global Python installation stays tidy.
  • Reproducibility: It makes it easier to replicate your project's environment elsewhere.

Here's how to create and activate one:

  1. Navigate to your project directory: Open your terminal or command prompt and cd into the folder where you want your Django project to live (or create a new folder).

    mkdir my_django_project
cd my_django_project

  2. Create the virtual environment: We'll use Python's built-in venv module. Let's call our environment env (a common convention).

    # On macOS/Linux
python3 -m venv env

# On Windows
python -m venv env

    This command creates a folder named env containing a copy of Python and pip specific to this project.

  3. Activate the virtual environment: You need to activate it each time you work on this project in a new terminal session.

    • On macOS/Linux:

      source env/bin/activate

      Your terminal prompt should now change, often showing (env) at the beginning, indicating the environment is active.

    • On Windows (Command Prompt):

      env\Scripts\activate

      Your command prompt line should now start with (env).

    • On Windows (PowerShell):

      .\env\Scripts\Activate.ps1

      (You might need to adjust your PowerShell execution policy first: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser)

Step 2: Install Django

Now that your virtual environment is active, installing Django is incredibly simple using pip:

pip install django

Pip will connect to the Python Package Index (PyPI), download the latest stable version of Django and its dependencies, and install them inside your active virtual environment.

You'll see output in your terminal showing the download and installation progress.

(Optional: To install a specific version of Django, you can specify it like this: pip install django==4.2)

Step 3: Verify the Installation

How do you know if it worked? You can quickly check the installed Django version. With your virtual environment still active, run:

django-admin --version

If Django installed correctly, this command will output the version number (e.g., 4.2.7).

Alternatively, you can start a Python interpreter and try importing Django:

  1. Type python (or python3) in your terminal to start the interpreter (you'll see >>>).
  2. Type import django and press Enter.
  3. Type print(django.get_version()) and press Enter.

If you see the version number and no errors, congratulations!

django-admin startproject myprojectname .

(Note the . at the end - it creates the project in the current directory).

From there, you can explore the official Django tutorial, learn about apps, models, views, templates, and start building amazing things.

Happy coding!
